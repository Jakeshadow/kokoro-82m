Deploy Kokoro-82M with Docker — Production TTS Server

OpenAI-compatible /v1/audio/speech endpoint. Pre-built images for CPU, NVIDIA GPU, AMD ROCm, and Apple Silicon. One docker run command to production.

Quick Start Docker Compose

Quick Start: One Command

Pick your hardware and run. Kokoro-FastAPI listens on port 8880 with an OpenAI-compatible endpoint.

CPU (Any Machine)

docker run -p 8880:8880 ghcr.io/remsky/kokoro-fastapi-cpu:latest

NVIDIA GPU

docker run --gpus all -p 8880:8880 ghcr.io/remsky/kokoro-fastapi-gpu:latest

AMD ROCm

docker run --device /dev/kfd --device /dev/dri -p 8880:8880 ghcr.io/remsky/kokoro-fastapi-rocm:latest

Apple Silicon (M1/M2/M3)

docker run -p 8880:8880 ghcr.io/remsky/kokoro-fastapi-mps:latest

Test the API

curl -X POST http://localhost:8880/v1/audio/speech \
  -H "Content-Type: application/json" \
  -d '{"model": "kokoro", "input": "Kokoro is running via FastAPI.", "voice": "af_heart"}' \
  --output speech.mp3

The API is OpenAI-compatible. Any client that works with OpenAI's TTS endpoint works with Kokoro-FastAPI — just change the base URL.

Docker Compose (Recommended for Production)

# docker-compose.yml
services:
  kokoro:
    image: ghcr.io/remsky/kokoro-fastapi-cpu:latest
    ports:
      - "8880:8880"
    environment:
      - DEFAULT_VOICE=af_heart
      - DEFAULT_LANG=a
    restart: unless-stopped
    volumes:
      - ./kokoro_model:/root/.cache/huggingface
docker compose up -d

GPU Compose Variant

services:
  kokoro-gpu:
    image: ghcr.io/remsky/kokoro-fastapi-gpu:latest
    ports:
      - "8880:8880"
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]
    environment:
      - DEFAULT_VOICE=af_heart
      - DEFAULT_LANG=a
    restart: unless-stopped

Fix HF 429 Errors: Local Model Caching

In cloud environments (GCP, AWS, Fly.io), every new container hits HuggingFace and triggers rate limits. Pre-download the model once and mount it as a volume.

Step 1: Download Model Files (Run Once)

pip install huggingface_hub
python -c "
from huggingface_hub import snapshot_download
snapshot_download('hexgrad/Kokoro-82M', local_dir='./kokoro_model')
"

Step 2: Mount in Docker

services:
  kokoro:
    image: ghcr.io/remsky/kokoro-fastapi-cpu:latest
    ports:
      - "8880:8880"
    environment:
      - HF_HOME=/data/models
      - HF_HUB_OFFLINE=1
    volumes:
      - ./kokoro_model:/data/models/hub
    restart: unless-stopped
HF_HUB_OFFLINE=1 prevents the container from ever calling HuggingFace. Combined with a cached model volume, your deployment is fully self-contained.

Common Docker Issues

Container starts but curl returns "connection refused"

The model takes 10–30 seconds to download on first run. Check logs:

docker logs kokoro

Wait for Uvicorn running on http://0.0.0.0:8880. With cached models, startup is under 5 seconds.

GPU not detected inside container

Check: docker run --gpus all nvidia/cuda:12.4.0-base-ubuntu22.04 nvidia-smi

If that fails, install NVIDIA Container Toolkit:

# Ubuntu
sudo apt-get install -y nvidia-container-toolkit
sudo systemctl restart docker
High memory usage / OOM

Kokoro-FastAPI loads the model into RAM. For CPU images, budget 2–3 GB. For GPU images, 2 GB VRAM. Add Docker memory limits:

services:
  kokoro:
    image: ghcr.io/remsky/kokoro-fastapi-cpu:latest
    deploy:
      resources:
        limits:
          memory: 4G
Audio quality differs from Python pipeline

Kokoro-FastAPI uses the same model weights but may apply different default post-processing. Check the response_format — wav is lossless. mp3 uses LAME encoding at 192kbps by default, which is transparent for speech.
