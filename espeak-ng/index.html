Fix espeak-ng Install Errors for Kokoro-82M

The #1 Kokoro setup blocker. Windows PATH, macOS Homebrew, Linux packages, and the no-install workaround.

Windows Fix Skip It (ONNX)

Why espeak-ng Blocks Kokoro

Kokoro uses espeak-ng as its phoneme generator — it converts text into phonemes (speech sounds) before the neural model turns them into audio. Without espeak-ng, Kokoro's Python pipeline crashes with OSError: espeak-ng not found or subprocess.CalledProcessError.

The GPL license on espeak-ng is the main reason community developers built the ONNX path — it replaces espeak-ng with a pure-ML phoneme encoder, avoiding both the install headache and the licensing concern. If you're stuck after 5 minutes, switch to ONNX.

Windows: espeak-ng Installation

Step 1: Download the Installer

Go to github.com/espeak-ng/espeak-ng/releases and download the latest .exe installer (e.g., espeak-ng-X64.msi).

Step 2: Install to a Known Location

Install to C:\Program Files\eSpeak NG\. The default location is fine — what matters is that both espeak-ng.exe and libespeak-ng.dll end up in the same directory.

Step 3: Add to System PATH (Most Common Fix)

This is what most people miss:

  1. Press Win + R, type sysdm.cpl
  2. Advanced → Environment Variables
  3. Under System variables, find Path → Edit
  4. Add: C:\Program Files\eSpeak NG
  5. Restart your terminal (PowerShell / CMD / VS Code terminal)
Restart is required. PATH changes don't apply to already-open terminals. Close and reopen.

Step 4: Verify

espeak-ng --version
# Should print: eSpeak NG version 1.51+

# Also test from Python:
python -c "import subprocess; subprocess.run(['espeak-ng', '--version'])"

macOS: espeak-ng Installation

brew install espeak-ng

If you get Error: espeak-ng has been disabled:

brew tap homebrew/core --force
brew install espeak-ng

Apple Silicon (M1/M2/M3) Extra Step

Set this environment variable for PyTorch compatibility:

export PYTORCH_ENABLE_MPS_FALLBACK=1

Add it to your ~/.zshrc to persist:

echo 'export PYTORCH_ENABLE_MPS_FALLBACK=1' >> ~/.zshrc

Verify

which espeak-ng
# Should print: /opt/homebrew/bin/espeak-ng  (Apple Silicon)
# or           /usr/local/bin/espeak-ng       (Intel)

espeak-ng "hello world" --stdout | aplay

The No-Install Workaround: Use ONNX

If you can't or don't want to install espeak-ng, the kokoro-onnx package replaces it entirely with a pure-ML phoneme encoder. No native dependencies. No GPL licensing concern.

pip install kokoro-onnx soundfile

# That's it. No espeak-ng needed.
from kokoro_onnx import Kokoro
kokoro = Kokoro("kokoro-v1.0.onnx", "voices.json")
samples, sample_rate = kokoro.create(
    "No espeak-ng was harmed in the making of this audio.",
    voice="af_sarah", speed=1.0
)
import soundfile as sf
sf.write("output.wav", samples, sample_rate)
