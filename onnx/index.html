Why ONNX Over Standard Kokoro

1. No espeak-ng

The ONNX version replaces espeak-ng with a pure-ML phoneme encoder. No native dependency to install. No GPL licensing concern. Works identically on Windows, macOS, and Linux.

2. Smaller & Faster on CPU

q4 quantized models are 305MB (vs 1.6GB for standard). ONNX Runtime's CPU execution provider is 2–2.5x faster than PyTorch on x86 CPUs. If you're running on a laptop without GPU, ONNX is the clear winner.

3. Runs in the Browser

kokoro-js + ONNX Runtime Web + Transformers.js = TTS entirely client-side. No server, no API costs, no GPU. Works on any modern browser. Ideal for client-side demos and privacy-sensitive apps.

4. Same Quality, Less Friction

The ONNX models are direct conversions of the original Kokoro weights. Same 54 voices, same 8 languages, same 24kHz output. The ML phoneme encoder is trained to match espeak-ng output exactly.

Python Setup (kokoro-onnx)

pip install kokoro-onnx soundfile
from kokoro_onnx import Kokoro

# Load model (auto-downloads on first run)
kokoro = Kokoro("kokoro-v1.0.onnx", "voices.json")

# Generate speech
samples, sample_rate = kokoro.create(
    "Life is like a box of chocolates — you never know what you're gonna get.",
    voice="af_sarah",
    speed=1.0
)

import soundfile as sf
sf.write("output.wav", samples, sample_rate)
print(f"Generated {len(samples)/sample_rate:.1f}s of audio")

Browser / JavaScript (kokoro-js)

Run Kokoro entirely in the browser via ONNX Runtime Web + Transformers.js. No server. No API key. Works offline after first load.

npm install kokoro-js
import { KokoroTTS } from "kokoro-js";

// Load model (q4: ~305MB download, cached after first load)
const tts = await KokoroTTS.from_pretrained(
  "onnx-community/Kokoro-82M-ONNX",
  { dtype: "q4" }
);

// Generate speech
const audio = await tts.generate(
  "Kokoro is running right here in your browser — no server, no API key, no latency.",
  { voice: "af_heart" }
);

// Play it
const blob = new Blob([audio.audioData], { type: "audio/wav" });
const url = URL.createObjectURL(blob);
new Audio(url).play();

Browser Demo Skeleton

<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"></head>
<body>
  <textarea id="text">Type something...</textarea>
  <select id="voice">
    <option value="af_heart">Heart (American Female)</option>
    <option value="am_adam">Adam (American Male)</option>
    <option value="bf_emma">Emma (British Female)</option>
  </select>
  <button id="speak">Speak</button>
  <audio id="player" controls></audio>

  <script type="module">
    import { KokoroTTS } from "https://cdn.jsdelivr.net/npm/kokoro-js/+esm";
    const tts = await KokoroTTS.from_pretrained(
      "onnx-community/Kokoro-82M-ONNX", { dtype: "q4" }
    );
    document.getElementById("speak").onclick = async () => {
      const text = document.getElementById("text").value;
      const voice = document.getElementById("voice").value;
      const result = await tts.generate(text, { voice });
      const blob = new Blob([result.audioData], { type: "audio/wav" });
      document.getElementById("player").src = URL.createObjectURL(blob);
    };
  </script>
</body>
</html>

First load: ~305MB model download (q4). Subsequent loads use browser cache. For production, host the model files on your own CDN and set KokoroTTS.from_pretrained("https://your-cdn.com/models", ...).

Docker: ONNX-Specific Image

The ONNX community maintains a Docker image with ONNX Runtime pre-installed:

docker run -p 8880:8880 ghcr.io/onnx-community/kokoro-onnx-server:latest
